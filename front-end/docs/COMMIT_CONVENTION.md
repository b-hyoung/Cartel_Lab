# 커밋 컨벤션

## 형식

```
<타입> : <제목>

<본문 (선택)>
```

---

## 타입

| 타입 | 설명 |
|---|---|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refact` | 기능 변경 없이 코드 구조 개선 |
| `style` | UI/스타일 변경 (기능 무관) |
| `chore` | 설정, 패키지, 빌드 관련 |
| `docs` | 문서 추가/수정 |
| `클린업` | 불필요한 코드/파일 제거 |
| `WIP` | 작업 중인 임시 커밋 (나중에 squash로 정리) |

---

## 규칙

- 타입 뒤에 ` : ` (공백 포함) 붙이기
- 제목은 한국어로, 명사형으로 마무리
- 제목은 50자 이내
- 본문은 **무엇을** 했는지보다 **왜** 했는지 위주로 작성

---

## 예시

```
feat : 헤더 UserDropdown 컴포넌트 추가

refact : 헤더 컴포넌트 분리 (Navbar / UserDropdown / MobileMenuToggle)

fix : 모바일 메뉴 외부 클릭 시 닫히지 않는 버그 수정

chore : next-auth, redux-toolkit 패키지 설치

클린업 : front-end 불필요한 기본 파일 제거

WIP : 헤더 스타일 작업 중
```

---

## WIP & Squash 전략

작업 중 임시 저장이 필요할 때 `WIP` 커밋을 사용합니다.
PR 올리기 전에 반드시 squash로 의미 단위로 정리합니다.

```bash
# 최근 N개 커밋을 하나로 합치기
git rebase -i HEAD~N
```

**흐름 예시**
```
WIP : 헤더 레이아웃 작업 중
WIP : 모바일 메뉴 추가 중
WIP : 드롭다운 스타일 조정
       ↓ squash
feat : 헤더 컴포넌트 마이그레이션 (Django → Next.js)
```
